# MINUTA DE RELEVAMIENTO

## 1. Introducción
Se lleva a cabo la presente minuta con el objetivo de relevar los requerimientos para el desarrollo de un sistema de gestión de changas o trabajos cortos. Este sistema permitirá que los empleadores publiquen oportunidades laborales de corta duración y que diversos trabajadores puedan postularse a dichas ofertas.

## 2. Usuarios del Sistema
- **Empleadores:** Personas o empresas que necesitan contratar trabajadores por un tiempo limitado.
- **Trabajadores:** Individuos en busca de oportunidades laborales de corta duración.
- **Administradores del sistema:** Responsables de la gestión y mantenimiento de la plataforma.

## 3. Objetivos del Sistema
- Permitir a los empleadores publicar changas con detalles como descripción del trabajo, remuneración, ubicación y requisitos.
- Facilitar la postulación de trabajadores a las ofertas publicadas.
- Implementar un sistema de notificación para alertar a los usuarios sobre nuevas oportunidades o cambios en sus postulaciones.
- Integrar un sistema de calificación para mejorar la confianza entre empleadores y trabajadores.
- Brindar herramientas de administración para la gestión de usuarios y ofertas laborales.

## 4. Funcionalidades del Sistema
- **Registro y autenticación de usuarios:** Creación de cuentas para empleadores y trabajadores con información básica (email, nombre de usuario, contraseña, teléfono).
- **Publicación de ofertas laborales:** Los empleadores pueden publicar trabajos con detalles como título, categoría, precio, fecha/hora, ubicación y descripción.
- **Postulaciones a trabajos:** Los empleados pueden postularse a las ofertas activas, y los empleadores pueden aceptar postulaciones.
- **Sistema de calificación:** Los empleadores pueden calificar a los trabajadores con puntuaciones de 1 a 5 y dejar comentarios.
- **Estado de postulaciones:** Seguimiento de postulaciones con estados definidos (pendiente, aceptado, rechazado).
- **Panel de administración:** Herramientas para la gestión de la plataforma y supervisión de contenido.

## 5. Requerimientos Técnicos
- Aplicación web responsiva sin versión móvil nativa.
- Base de datos para almacenamiento de usuarios, trabajos y postulaciones.
- Seguridad en el manejo de datos personales.

## 6. Consideraciones Finales
El sistema busca ser una solución eficiente para conectar empleadores con trabajadores de manera rápida y sencilla, optimizando el proceso de contratación para trabajos de corta duración. Se prevén futuras revisiones para ajustar detalles y validar la viabilidad de los requerimientos.
